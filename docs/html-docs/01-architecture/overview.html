Architecture - Vue d'ensemble

FitMyCV.io est une application web full-stack monolithique construite avec Next.js 16 (App Router). Elle permet aux utilisateurs de créer, optimiser et exporter des CV personnalisés grâce à l'intelligence artificielle.

Caractéristiques Architecturales

Aspect Choix technologique
Type Monolith full-stack
Framework Next.js 16.1.1 (App Router)
Bundler Turbopack (défaut Next.js 16)
Rendering Hybrid (SSR + Client Components)
API API Routes (serverless-like)
Base de données PostgreSQL via Prisma 6
Authentification NextAuth.js 4 (OAuth + Credentials)
Paiements Stripe (abonnements + crédits)
IA OpenAI API (modèles configurables)
Extension Chrome/Firefox Manifest V3

Organisation des Modules Métier

Le dossier lib/ contient 34 modules organisés par domaine fonctionnel :

Authentification & Sécurité

  • auth/ - NextAuth 4 config, session, extension token
  • security/ - Validation fichiers, politique mots de passe, XSS
  • recaptcha/ - Protection bot
  • cookies/ - Consent RGPD, registre cookies

CV Core

  • cv-core/ - Storage DB, validation, versioning, review
  • job-offer/ - Extraction structurée offres (JSON)
  • scoring/ - Match score CV/offre
  • translation/ - Traduction CV (4 langues)
  • export/ - Templates et sélections export

Intelligence Artificielle

  • openai-core/ - Client API, schema loader, prompt loader
  • features/cv-adaptation/ - Pipeline adaptation CV
  • features/cv-improvement/ - Pipeline optimisation
  • features/import-pdf/ - Import PDF via IA
  • features/template-generation/ - Génération templates
  • features/job-title-generation/ - Génération depuis intitulé
  • prompts-shared/ - Prompts et règles partagés

Billing & Abonnements

  • subscription/ - Plans, crédits, limites, Stripe sync
  • stripe/ - Checkout locale
  • creditCosts/ - Context coûts crédits

Async Processing

  • background-jobs/ - Queue, runner, process registry
  • events/ - Event emitter DB temps réel
  • loading/ - Événements de chargement

Communication & Monitoring

  • email/ - SMTP + Resend, templates, triggers
  • telemetry/ - Tracking, OpenAI usage, analytics
  • i18n/ - Labels CV, language context
  • onboarding/ - Steps, state, config

Points Clés de l'Architecture

Monolith Modulaire

Bien que monolithique, l'architecture est fortement modulaire avec une séparation claire des responsabilités. Chaque module dans lib/ est autonome et expose une API publique via index.js.

5 Pipelines IA Spécialisés

Le dossier lib/features/ contient 5 pipelines IA indépendants : adaptation CV, amélioration CV, import PDF, génération depuis intitulé de poste, et génération de templates. Chaque pipeline utilise des appels OpenAI parallélisés avec structured output (JSON Schema).

Background Jobs In-Process

La queue de jobs est gérée en mémoire (pas de Redis/RabbitMQ). Cela simplifie le déploiement mais limite la scalabilité horizontale. Le processRegistry gère la concurrence des tâches.
