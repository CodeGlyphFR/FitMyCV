Architecture - Vue d'ensemble

FitMyCV.io est une application web full-stack monolithique construite avec Next.js 16 (App Router). Elle permet aux utilisateurs de créer, optimiser et exporter des CV personnalisés grâce à l'intelligence artificielle.

Caractéristiques Architecturales

Aspect Choix technologique
Type Monolith full-stack
Framework Next.js 16.1.1 (App Router)
Bundler Turbopack (défaut Next.js 16)
Rendering Hybrid (SSR + Client Components)
API API Routes (serverless-like)
Base de données PostgreSQL via Prisma 6
Authentification NextAuth.js 4 (OAuth + Credentials)
Paiements Stripe (abonnements + crédits)
IA OpenAI API (modèles configurables)

Organisation des Modules Métier

Le dossier lib/ contient 31 modules organisés par domaine fonctionnel :

Authentification

  • auth/ - NextAuth config + session
  • security/ - Validation, sanitization
  • recaptcha/ - Protection bot

CV Core

  • cv-core/ - Storage, validation, versioning
  • job-offer/ - Extraction offres
  • scoring/ - Match score CV/offre
  • translation/ - Traduction CV

Intelligence Artificielle

  • openai-core/ - Client API, helpers
  • features/cv-adaptation/ - Pipeline génération
  • features/cv-improvement/ - Pipeline optimisation
  • prompts-shared/ - Prompts réutilisables

Billing

  • subscription/ - Plans, crédits, limites

Async Processing

  • background-jobs/ - Queue tâches (max 3 concurrent)

Communication

  • email/ - SMTP + Resend
  • events/ - Event emitter temps réel

Points Clés de l'Architecture

Monolith Modulaire

Bien que monolithique, l'architecture est fortement modulaire avec une séparation claire des responsabilités. Chaque module dans lib/ est autonome et expose une API publique via index.js.

Pipelines IA Parallélisés

Les pipelines de génération et d'optimisation CV utilisent une architecture par phases avec parallélisation (max 5 appels OpenAI concurrents) et cache intelligent pour optimiser les coûts.

Background Jobs In-Process

La queue de jobs est gérée en mémoire (pas de Redis/RabbitMQ). Cela simplifie le déploiement mais limite la scalabilité horizontale. Max 3 jobs globaux en parallèle.
