Credentials Flow

L'authentification par credentials permet aux utilisateurs de se connecter avec leur email et mot de passe.

Flux d'Inscription

Processus d'inscription
sequenceDiagram participant U as Utilisateur participant F as Frontend participant A as API /register participant DB as PostgreSQL U->>F: Remplit formulaire F->>A: POST { email, password, name } A->>A: Validation Zod A->>DB: Vérifie email unique A->>A: Hash bcrypt (10 rounds) A->>DB: INSERT User A->>DB: INSERT CreditBalance A->>F: 201 Created F->>U: Redirection login

Flux de Connexion

Processus de connexion
sequenceDiagram participant U as Utilisateur participant F as Frontend participant NA as NextAuth participant DB as PostgreSQL U->>F: Email + Password F->>NA: signIn('credentials', {...}) NA->>DB: SELECT User by email NA->>NA: bcrypt.compare() alt Password valide NA->>NA: Génère JWT NA->>F: Session créée F->>U: Redirection dashboard else Password invalide NA->>F: Error: Invalid credentials F->>U: Affiche erreur end

Hachage des Mots de Passe

// lib/auth/password.js
import bcrypt from 'bcrypt';

const SALT_ROUNDS = 10;

export async function hashPassword(password) {
  return bcrypt.hash(password, SALT_ROUNDS);
}

export async function verifyPassword(password, hashedPassword) {
  return bcrypt.compare(password, hashedPassword);
}

Validation des Donnees

// Schema Zod pour l'inscription
import { z } from 'zod';

export const registerSchema = z.object({
  email: z.string().email('Email invalide'),
  password: z.string()
    .min(8, 'Minimum 8 caracteres')
    .regex(/[A-Z]/, 'Au moins une majuscule')
    .regex(/[0-9]/, 'Au moins un chiffre'),
  name: z.string().min(2, 'Nom requis')
});

// Schema pour la connexion
export const loginSchema = z.object({
  email: z.string().email('Email invalide'),
  password: z.string().min(1, 'Mot de passe requis')
});

API Endpoints

Endpoint Méthode Description
/api/auth/register POST Inscription nouvel utilisateur
/api/auth/callback/credentials POST Connexion (via NextAuth)
/api/auth/forgot-password POST Demande reset password
/api/auth/reset-password POST Reset avec token

Securite

Mesures de securite
  • Hachage bcrypt avec 10 rounds
  • Validation stricte des inputs (Zod)
  • Rate limiting sur /register et /login
  • Tokens de reset password avec expiration (1h)
