Verification Email

Le système de verification email permet de confirmer l'adresse email des utilisateurs inscrits par credentials.

Flux de Verification

Processus de verification
sequenceDiagram participant U as Utilisateur participant A as API participant DB as PostgreSQL participant E as Email Service Note over U,E: Inscription U->>A: POST /register A->>DB: Creer User (emailVerified: null) A->>DB: Creer VerificationToken A->>E: Envoyer email verification E->>U: Email avec lien Note over U,E: Verification U->>A: GET /verify?token=xxx A->>DB: Verifier token valide A->>DB: UPDATE User.emailVerified A->>DB: DELETE VerificationToken A->>U: Redirection login

Modèle VerificationToken

// prisma/schema.prisma

model VerificationToken {
  identifier String   // Email de l'utilisateur
  token      String   @unique
  expires    DateTime

  @@unique([identifier, token])
}

Generation du Token

// lib/auth/verification.js
import crypto from 'crypto';
import prisma from '@/lib/prisma';

export async function createVerificationToken(email) {
  // Generer token unique
  const token = crypto.randomBytes(32).toString('hex');

  // Expiration dans 24h
  const expires = new Date(Date.now() + 24 * 60 * 60 * 1000);

  // Sauvegarder en base
  await prisma.verificationToken.create({
    data: {
      identifier: email,
      token,
      expires
    }
  });

  return token;
}

export async function verifyToken(token) {
  const record = await prisma.verificationToken.findUnique({
    where: { token }
  });

  if (!record || record.expires < new Date()) {
    return null;
  }

  // Marquer l'email comme vérifie
  await prisma.user.update({
    where: { email: record.identifier },
    data: { emailVerified: new Date() }
  });

  // Supprimer le token utilise
  await prisma.verificationToken.delete({
    where: { token }
  });

  return record.identifier;
}

API Endpoints

Endpoint Méthode Description
/api/auth/verify GET Vérifie le token et active le compte
/api/auth/resend-verification POST Renvoie l'email de verification

Securite

Mesures de securite
  • Token cryptographiquement securise (32 bytes)
  • Expiration automatique apres 24h
  • Token a usage unique (supprimé apres verification)
  • Rate limiting sur resend (1 par minute)
