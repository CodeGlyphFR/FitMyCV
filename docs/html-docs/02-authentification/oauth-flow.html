OAuth Flow

FitMyCV.io supporte l'authentification OAuth 2.0 via Google et LinkedIn, permettant une connexion rapide et securisee.

Flux OAuth 2.0

Sequence OAuth
sequenceDiagram participant U as Utilisateur participant F as FitMyCV participant P as Provider (Google/LinkedIn) participant DB as PostgreSQL U->>F: Clic "Connexion avec Google" F->>P: Redirection vers OAuth P->>U: Page de consentement U->>P: Autorise l'acces P->>F: Code d'autorisation F->>P: Echange code contre token P->>F: Access token + profil F->>DB: Creer/mettre a jour User F->>DB: Creer Account (liaison) F->>U: Session créée, redirection

Providers Configures

Google OAuth

  • Scopes: email, profile
  • Donnees recuperees: email, nom, avatar
  • Callback: /api/auth/callback/google

LinkedIn OAuth

  • Scopes: r_liteprofile, r_emailaddress
  • Donnees recuperees: email, nom, photo
  • Callback: /api/auth/callback/linkedin

Configuration Provider

// lib/auth/options.js

import GoogleProvider from 'next-auth/providers/google';
import LinkedInProvider from 'next-auth/providers/linkedin';

export const authOptions = {
  providers: [
    GoogleProvider({
      clientId: process.env.GOOGLE_CLIENT_ID,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET,
      authorization: {
        params: {
          prompt: 'consent',
          access_type: 'offline',
          response_type: 'code'
        }
      }
    }),

    LinkedInProvider({
      clientId: process.env.LINKEDIN_CLIENT_ID,
      clientSecret: process.env.LINKEDIN_CLIENT_SECRET,
      authorization: {
        params: {
          scope: 'r_liteprofile r_emailaddress'
        }
      }
    })
  ]
};

Liaison de Comptes

Quand un utilisateur se connecte via OAuth, NextAuth crée automatiquement :

Table Donnees
User id, email, name, image (si nouveau)
Account provider, providerAccountId, access_token, refresh_token

Gestion Multi-Providers

Un utilisateur peut lier plusieurs providers a son compte :

// Verifier les comptes lies
const accounts = await prisma.account.findMany({
  where: { userId: session.user.id }
});
