Sécurité du Code

FitMyCV intègre trois modules de sécurité dans lib/security/ : la validation des fichiers uploadés (magic numbers, MIME, taille), un logger sécurisé qui masque automatiquement les données personnelles (PII), et une couche de sanitisation XSS avec whitelist HTML stricte.

validateUploadedFile(file, options)

import { validateUploadedFile } from '@/lib/security/fileValidation';

const result = await validateUploadedFile(file, {
  allowedTypes: ['application/pdf'],  // Types MIME autorisés
  maxSize: 10 * 1024 * 1024,          // 10 MB (optionnel)
});

if (result.valid) {
  // result.buffer contient le Buffer du fichier
  await saveFile(result.buffer);
} else {
  // result.error contient le message d'erreur
  throw new Error(result.error);
}

Vérification Magic Numbers

Le type MIME déclaré par le navigateur peut être falsifié. La validation vérifie les premiers octets du fichier (magic numbers) pour confirmer le type réel :

Type Magic Number (hex) Représentation ASCII Taille max
application/pdf 25 50 44 46 %PDF 10 MB
Autres types Extensible — 5 MB (défaut) 
// Signatures internes
const FILE_SIGNATURES = {
  pdf: [
    [0x25, 0x50, 0x44, 0x46], // %PDF
  ],
};

// Vérification : les 4 premiers octets doivent correspondre
function verifyMagicNumber(buffer, expectedType) {
  const type = expectedType.split('/')[1]; // 'application/pdf' → 'pdf'
  const signatures = FILE_SIGNATURES[type];
  return signatures.some(sig =>
    sig.every((byte, i) => buffer[i] === byte)
  );
}

sanitizeFilename(filename)

Nettoie un nom de fichier pour prévenir les injections :

import { sanitizeFilename } from '@/lib/security/fileValidation';

sanitizeFilename('rapport final.pdf');
// → "rapport_final_pdf"

sanitizeFilename('../../../etc/passwd');
// → "etc_passwd"

sanitizeFilename('..hidden');
// → "hidden"

sanitizeFilename('');
// → "file_1708099200000"  (nom généré avec timestamp)
Règle Action
Caractères non alphanumériques Remplacés par _
Séquences de points (..) Réduites à un seul .
Points en début de nom Supprimés (prévention fichiers cachés)
Longueur maximale Tronqué à 255 caractères

isPathSafe(filepath)

Détecte les tentatives de path traversal :

import { isPathSafe } from '@/lib/security/fileValidation';

isPathSafe('uploads/cv/2024/report.pdf');  // → true
isPathSafe('../../../etc/passwd');          // → false (.. détecté)
isPathSafe('~/secret/file.txt');           // → false (~/ détecté)
isPathSafe('path//to//file');              // → false (// détecté)
isPathSafe('path\\to\\file');              // → false (\\ détecté)
Patterns dangereux détectés — La fonction bloque 4 patterns : .. (traversal parent), ~/ (home directory), // (double slash), \ (backslash Windows).

Logger Sécurisé — Masquage PII

Le module secureLogger.js remplace console.log par des fonctions qui masquent automatiquement les données personnelles identifiables (PII) avant affichage dans les logs.

Types de données masquées

Donnée Pattern détecté Résultat masqué
Email user@domain.com u***r@domain.com
Clé API sk-proj-abc123... [API_KEY_HIDDEN]
Token Bearer bearer eyJhbGci... bearer [TOKEN_HIDDEN]
Mot de passe "password":"secret" "password":"[HIDDEN]"
Token JSON "token":"eyJ..." "token":"[HIDDEN]"
User ID userId: "clx9abc..." userId: "usr_a1b2c3" (hash SHA-256)

API du logger

import logger from '@/lib/security/secureLogger';

// Remplacement direct de console.*
logger.log('User login:', { email: 'jean@example.com', token: 'secret' });
// Console: User login: { email: "j***n@example.com", token: "[HIDDEN]" }

logger.warn('API call failed', { apiKey: 'sk-proj-abc123def456' });
// Console: API call failed { apiKey: "[API_KEY_HIDDEN]" }

logger.error('Auth error', 'bearer eyJhbGciOiJIUzI1NiJ9...');
// Console: Auth error bearer [TOKEN_HIDDEN]

// Logger avec contexte
logger.context('auth', 'warn', 'Failed login for', 'user@test.com');
// Console: [auth] Failed login for u***r@test.com

Fonctions exportées

Fonction Équivalent Description

Traitement récursif des objets

La fonction sanitizeData() parcourt récursivement les objets et applique les règles de masquage selon le nom de la clé :

import { sanitizeData } from '@/lib/security/secureLogger';

const data = {
  user: {
    userId: 'clx9abc123def456',
    email: 'jean.dupont@gmail.com',
    password: 'SuperSecret123',
    profile: {
      name: 'Jean Dupont',
      token: 'eyJhbGciOiJIUzI1NiJ9'
    }
  }
};

sanitizeData(data);
// → {
//   user: {
//     userId: "usr_a1b2c3",
//     email: "j***t@gmail.com",
//     password: "[HIDDEN]",
//     profile: {
//       name: "Jean Dupont",
//       token: "[HIDDEN]"
//     }
//   }
// }
Clés masquées automatiquement — Les clés suivantes sont entièrement remplacées par [HIDDEN] : password, token, apiKey, secret, passwordHash.

escapeHtml(str)

Échappe les 6 caractères HTML dangereux :

import { escapeHtml } from '@/lib/security/xssSanitization';

escapeHtml('<script>alert("XSS")</script>');
// → "&lt;script&gt;alert(&quot;XSS&quot;)&lt;/script&gt;"

escapeHtml("L'utilisateur & son profil");
// → "L&#x27;utilisateur &amp; son profil"

sanitizeHtml(html) — Whitelist

Autorise uniquement les balises de mise en forme basique et supprime tout le reste :

// Balises autorisées (whitelist)
const ALLOWED_TAGS = ['b', 'i', 'em', 'strong', 'u', 'br', 'p', 'ul', 'ol', 'li'];

// Aucun attribut autorisé (sécurité maximale)
const ALLOWED_ATTRIBUTES = {};

La sanitisation se fait en 4 passes :

Passe Action Exemple
1 Suppression des blocs <script> et <style> <script>alert(1)</script> →
2 Suppression des URLs javascript: javascript:alert(1) → alert(1)
3 Suppression des event handlers (onclick, onload...) onclick="hack()" →
4 Filtrage des balises (whitelist) + suppression des attributs <div class="x"> → , <b> → <b>

sanitizeCvData(cvData)

Fonction spécialisée pour les données de CV. Les CVs étant des données structurées en JSON, tout le HTML est supprimé (stratégie stripHtml) :

import { sanitizeCvData } from '@/lib/security/xssSanitization';

const dirtyData = {
  name: 'Jean <script>alert(1)</script> Dupont',
  experience: [
    {
      title: 'Dev <b onclick="hack()">Full-Stack</b>',
      description: '<p>Développement</p> <img src=x onerror=alert(1)>'
    }
  ]
};

sanitizeCvData(dirtyData);
// → {
//   name: "Jean  Dupont",
//   experience: [{
//     title: "Dev Full-Stack",
//     description: "Développement "
//   }]
// }

sanitizeUrl(url)

Valide et sanitise les URLs en vérifiant le protocole :

import { sanitizeUrl } from '@/lib/security/xssSanitization';

sanitizeUrl('https://github.com/user');     // → "https://github.com/user"
sanitizeUrl('http://example.com');           // → "http://example.com"
sanitizeUrl('javascript:alert(1)');          // → null (protocole bloqué)
sanitizeUrl('data:text/html,...');            // → null (protocole bloqué)
sanitizeUrl('ftp://files.com/cv.pdf');       // → null (non autorisé)
sanitizeUrl('not-a-url');                    // → null (invalide)
Protocoles autorisés — Seuls http: et https: sont acceptés par défaut. Les protocoles javascript:, data: et vbscript: sont explicitement bloqués.

sanitizeEmail(email)

Valide le format et nettoie les caractères dangereux :

import { sanitizeEmail } from '@/lib/security/xssSanitization';

sanitizeEmail('Jean.Dupont@Gmail.COM');       // → "jean.dupont@gmail.com"
sanitizeEmail('  user@test.fr  ');            // → "user@test.fr"
sanitizeEmail('user<script>@evil.com');  // → null (caractères dangereux)
sanitizeEmail('not-an-email');                // → null (format invalide)

Récapitulatif des protections

Menace Protection Module
Upload de fichier malveillant Magic numbers + MIME + taille fileValidation.js
Path traversal (../../etc/passwd) isPathSafe() + sanitizeFilename() fileValidation.js
Fuite de PII dans les logs Masquage automatique emails, tokens, IDs secureLogger.js
XSS stocké (données CV) sanitizeCvData() strip complet xssSanitization.js

Défense en profondeur — Ces trois modules forment une couche de sécurité applicative qui complète les protections au niveau middleware (CORS, CSP, rate limiting) et infrastructure (HTTPS, WAF).
