CRUD CV

Operations de creation, lecture, mise a jour et suppression des CV. Les endpoints utilisent le pattern filename-based (pas d'ID numerique). Authentification via auth() (next-auth session).

Creation d'un CV

POST /api/cvs/create — app/api/cvs/create/route.js

INPUT Body JSON 
{
  "full_name": "Jean Dupont",
  "current_title": "Developpeur Full-Stack",
  "email": "jean@example.com",
  "recaptchaToken": "03AGdBq24..."
}

Etapes du traitement

  1. Authentification : Verification de la session via auth()
  2. reCAPTCHA : Validation du token via verifyRecaptcha(recaptchaToken, { callerName: 'create-cv', scoreThreshold: 0.5 })
  3. Limites d'utilisation : Verification et incrementation du compteur via incrementFeatureCounter(userId, 'create_cv_manual')
  4. Creation du JSON : Structure vide avec 8 sections
  5. Generation du filename : {timestamp}.json (auto-genere, unicite verifiee)
  6. Ecriture en DB : Via writeUserCvFile() puis prisma.cvFile.upsert()
  7. Cookie : Definition du cookie cvFile pour selection immediate
  8. Telemetrie : Tracking via trackCvCreation()

Structure CV vide (8 sections)

// app/api/cvs/create/route.js

var cv = {
  header: { full_name, current_title, contact: { email, links: [], location: {} } },
  summary: { description: "" },
  skills: { hard_skills: [], tools: [], methodologies: [], soft_skills: [] },
  experience: [],
  education: [],
  languages: [],

  projects: []
};
Filename unique

Le nom de fichier est genere a partir de Date.now(). Si un fichier du meme nom existe deja, le timestamp est incremente jusqu'a obtenir un nom unique.

Selection du CV courant

Le champ current est determine depuis le cookie cvFile. Si le cookie ne correspond a aucun fichier existant, le premier fichier de la liste est utilise.

// app/api/cvs/read/route.js

export async function GET(request) {
  const session = await auth();
  if (!session?.user?.id) {
    return CommonErrors.notAuthenticated();
  }

  const { searchParams } = new URL(request.url);
  const filename = searchParams.get('file');

  if (!filename) {
    return CvErrors.missingFilename();
  }

  const cvContent = await readUserCvFile(session.user.id, filename);
  const cv = JSON.parse(cvContent);

  return NextResponse.json({ cv });
}

Suppression

POST /api/cvs/delete — app/api/cvs/delete/route.js

INPUT Body JSON 
{
  "file": "1234567890.json"
}

Etapes du traitement

  1. Authentification : Verification session via auth()
  2. Validation filename : Pas de / ni \, doit finir par .json (fonction isJsonFileSafe())
  3. Verification existence : Via cvFileExists(userId, file)
  4. Suppression : Via deleteUserCvFile() — cascade Prisma supprime les versions associees
  5. Selection suivante : Choix du prochain CV via pickNextFile() (le plus recent par createdAt DB)
  6. Telemetrie : Tracking via trackEvent(EventTypes.CV_DELETED)
// app/api/cvs/delete/route.js

function isJsonFileSafe(name) {
  if (typeof name !== 'string') return false;
  if (name.includes('/') || name.includes('\\')) return false;
  return name.toLowerCase().endsWith('.json');
}

async function pickNextFile(userId) {
  const nextCv = await prisma.cvFile.findFirst({
    where: { userId },
    orderBy: { createdAt: 'desc' },
    select: { filename: true },
  });
  return nextCv?.filename || null;
}
OUTPUT Reponse JSON 
{
  "ok": true,
  "nextFile": "9876543210.json"
}

Suppression en masse

POST /api/cvs/delete-bulk — app/api/cvs/delete-bulk/route.js

INPUT Body JSON 
{
  "files": ["1234567890.json", "9876543210.json"]
}

Contraintes

  • Minimum : 1 fichier requis (sinon erreur NO_FILES)
  • Maximum : 50 fichiers par requete (sinon erreur TOO_MANY_FILES)
  • Validation : Chaque filename passe par isJsonFileSafe()
  • Non-bloquant : Si un fichier echoue, les autres continuent
OUTPUT Reponse JSON 
{
  "ok": true,
  "deletedCount": 2,
  "errors": undefined
}

Le champ errors contient la liste des filenames en echec (present uniquement si au moins un fichier a echoue).

Storage Layer

lib/cv-core/storage.js — Implementation DB via Prisma (migration depuis filesystem).

Architecture

Toutes les operations passent par PostgreSQL via Prisma. Les signatures des fonctions sont conservees pour compatibilite avec le reste du codebase (migration depuis filesystem). Le contenu JSON est stocke dans la colonne content (type Json) de la table CvFile.

writeUserCvFile(userId, filename, content)

Ecrit ou met a jour le contenu d'un CV via Prisma upsert. Sanitize les null bytes via removeNullBytes() (PostgreSQL ne supporte pas \u0000 dans les colonnes JSON).

// lib/cv-core/storage.js

export async function writeUserCvFile(userId, filename, content) {
  const rawData = typeof content === 'string' ? JSON.parse(content) : content;
  // Sanitize null bytes - PostgreSQL doesn't support \u0000 in text/json columns
  const cvData = removeNullBytes(rawData);

  await prisma.cvFile.upsert({
    where: { userId_filename: { userId, filename } },
    update: {
      content: cvData,
      updatedAt: new Date(),
    },
    create: {
      userId,
      filename,
      content: cvData,
      contentVersion: 1,
    },
  });

  return `db://users/${userId}/cvs/${filename}`;
}

readUserCvFile(userId, filename)

Retourne le contenu du CV sous forme de string JSON (JSON.stringify(content)).

// lib/cv-core/storage.js

export async function readUserCvFile(userId, filename) {
  const cvFile = await prisma.cvFile.findUnique({
    where: { userId_filename: { userId, filename } },
    select: { content: true },
  });

  if (!cvFile?.content) {
    throw new Error(`CV file not found: ${filename}`);
  }

  return JSON.stringify(cvFile.content);
}

deleteUserCvFile(userId, filename)

Supprime un CV. Gere l'erreur Prisma P2025 (record not found) en retournant false au lieu de throw.

// lib/cv-core/storage.js

export async function deleteUserCvFile(userId, filename) {
  try {
    await prisma.cvFile.delete({
      where: { userId_filename: { userId, filename } },
    });
    return true;
  } catch (error) {
    // P2025 = Record not found
    if (error.code === 'P2025') {
      return false;
    }
    throw error;
  }
}

listUserCvFiles(userId)

Retourne la liste des noms de fichiers, triee par createdAt descendant.

// lib/cv-core/storage.js

export async function listUserCvFiles(userId) {
  const files = await prisma.cvFile.findMany({
    where: { userId },
    select: { filename: true },
    orderBy: { createdAt: 'desc' },
  });
  return files.map((f) => f.filename);
}

Erreurs centralisees

Les reponses d'erreur sont gerees par les helpers CommonErrors et CvErrors depuis lib/api/apiErrors.js, garantissant un format de reponse uniforme sur tous les endpoints CV.
