Export

Système d'export de CV en PDF (Puppeteer) et DOCX (bibliothèque docx), avec un pipeline de génération HTML partagé, des templates de sélection persistés côté serveur, et un mécanisme de remboursement automatique des crédits en cas d'erreur.

Composants principaux

lib/pdf/cvUtils.js

Fonctions de préparation des données CV : prepareCvData(), formatDate(), formatLocation(), filterItems(), sortExperiences(), sortEducation(). Définit aussi DEFAULT_SECTION_ORDER (7 sections).

lib/pdf/cvStyles.js

Styles CSS pour l'export : getBaseStyles() (partagés), getExportStyles() (avec page-break CSS pour PDF), getPreviewStyles() (pour la prévisualisation en ligne).

lib/pdf/sectionGenerators.js

Génération HTML des sections : generateHeaderSection() pour l'en-tête, createSectionGenerators() retourne des générateurs pour 7 sections, generateSectionsHtml() assemble le tout.

lib/pdf/cvTranslations.js

Traductions des libellés CV en 4 langues (fr, en, es, de) : getTranslation(), translateLevel(), getSectionTitle(), createTranslator().

Sections du CV

L'ordre par défaut est défini dans DEFAULT_SECTION_ORDER :

# Section Clé Filtrage
— En-tête (nom, titre, contact) header Toujours rendu séparément par generateHeaderSection()
1 Résumé professionnel summary isSectionEnabled()
2 Compétences skills filterItems()
3 Expériences professionnelles experience filterItems() + sortExperiences()
4 Formation education filterItems() + sortEducation()
5 Langues languages filterItems()
6 Projets projects filterItems()
7 Extras extras filterItems()

Export Storage (client)

Le module lib/export/exportStorage.js gère la persistance locale des préférences d'export par CV via localStorage.

Selection Helpers

Le module lib/export/selectionHelpers.js fournit :

  • getDefaultSelections() — Sélections par défaut (toutes les sections activées)
  • migrateSelections() — Migration des anciennes sélections vers le nouveau format
  • calculateCounters() — Calcul des compteurs d'éléments sélectionnés
  • extractMacroSelections() — Extraction des sélections au niveau macro (section)

Gestion des crédits

Les deux endpoints d'export (PDF et DOCX) suivent le même pattern de gestion des crédits :

// 1. Débit avant l'export
const usageResult = await incrementFeatureCounter(userId, featureName);
const { transactionId, usedCredit } = usageResult;

try {
  // 2. Génération du fichier (PDF ou DOCX)
  const buffer = await generateExport(html);
  return new Response(buffer, { headers: { ... } });
} catch (error) {
  // 3. Remboursement automatique en cas d'erreur
  if (usedCredit && transactionId) {
    await refundCredit(userId, transactionId, `Échec export: ${error.message}`);
  }
  throw error;
}
Remboursement automatique — Si l'export échoue après le débit des crédits, la fonction refundCredit(userId, transactionId, message) rembourse automatiquement le crédit consommé.

prepareCvData() — Préparation des données

// lib/pdf/cvUtils.js (extraits clés)

export const DEFAULT_SECTION_ORDER = [
  'summary', 'skills', 'experience', 'education',
  'languages', 'projects', 'extras'
];

export function prepareCvData(cvData, selections) {
  const header = cvData.header || {};
  const contact = header.contact || {};
  const summary = cvData.summary || {};
  const section_titles = cvData.section_titles || {};

  // Trier et filtrer les données
  let experience = sortExperiences(cvData.experience || []);
  experience = filterItems(experience, selections, 'experience');

  let education = sortEducation(cvData.education || []);
  education = filterItems(education, selections, 'education');

  let projects = filterItems(cvData.projects || [], selections, 'projects');
  let skills = cvData.skills || {};
  let languages = filterItems(cvData.languages || [], selections, 'languages');
  let extras = filterItems(cvData.extras || [], selections, 'extras');

  return { header, contact, summary, experience, education,
           projects, skills, languages, extras, section_titles };
}

Assemblage HTML final (export-pdf)

// app/api/export-pdf/route.js — generateCvHtml()

function generateCvHtml(cvData, language = 'fr', selections = null, sectionsOrder = null) {
  const t = (path) => getTranslation(language, path);
  const order = sectionsOrder || DEFAULT_SECTION_ORDER;

  // Préparer les données
  const data = prepareCvData(cvData, selections);

  // Créer les générateurs de sections
  const sectionGenerators = createSectionGenerators(data, language, selections, {
    isExport: true, shouldBreakBefore: () => false,
  });

  // Générer le header
  const headerHtml = generateHeaderSection(
    data.header, data.contact, selections, language);

  // Générer les sections dans l'ordre
  const sectionsHtml = generateSectionsHtml(sectionGenerators, order);

  return `<!DOCTYPE html>
    <html lang="${language}">
    <head><style>${getExportStyles()}</style></head>
    <body>
      <div class="cv-container">
        ${headerHtml}
        ${sectionsHtml}
      </div>
    </body></html>`;
}
