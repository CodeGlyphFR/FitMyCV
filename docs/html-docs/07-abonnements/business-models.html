Modèles Économiques

FitMyCV.io implémente deux modèles économiques distincts, configurables via l'interface d'administration. Le choix du modèle affecte la façon dont les utilisateurs accèdent aux fonctionnalités.

Paramètre de configuration

Le mode actif est contrôlé par le setting subscription_mode_enabled dans la table Setting. Valeur : "true" ou "false".

Caractéristiques

Plans disponibles

  • Free : Limites strictes, fonctionnalités de base
  • Pro : Limites étendues, toutes les features
  • Premium : Illimité sur la plupart des features

Compteurs mensuels

  • Reset automatique à chaque période
  • Basé sur date anniversaire Stripe
  • Stocké dans FeatureUsageCounter

Extension par crédits

  • Si limite atteinte → utiliser crédits
  • Achat de packs (5, 10, 25 crédits)
  • Les crédits n'expirent pas

Logique de vérification

// lib/subscription/featureUsage.js - canUseFeature()

async function canUseFeature(userId, featureName) {
  // 1. Vérifier si balance négative (chargeback)
  if (balance < 0) return { canUse: false, reason: 'balance_negative' };

  // 2. Récupérer le plan de l'utilisateur
  const plan = await getUserPlan(userId);
  const limit = await getFeatureLimit(plan.id, featureName);

  // 3. Feature désactivée pour ce plan ?
  if (!limit.isEnabled) {
    // Peut utiliser crédits si disponibles
    return checkCreditsAvailable(userId, featureName);
  }

  // 4. Limite illimitée (-1) ?
  if (limit.usageLimit === -1) {
    return { canUse: true, useCredit: false };
  }

  // 5. Vérifier compteur mensuel
  const counter = await getFeatureCounter(userId, featureName);
  if (counter.count < limit.usageLimit) {
    return { canUse: true, useCredit: false };
  }

  // 6. Limite atteinte → vérifier crédits
  return checkCreditsAvailable(userId, featureName);
}

Caractéristiques

Pas d'abonnement Stripe

  • Aucune facturation récurrente
  • Pas de gestion de plans
  • Modèle "pay-as-you-go"

Crédits de bienvenue

  • Attribués à l'inscription
  • Montant configurable en admin
  • Permettent de tester le service

Packs de crédits

  • Achat ponctuel via Stripe Checkout
  • Crédités immédiatement
  • Prix et quantités configurables

Logique de vérification (mode crédits)

// lib/subscription/featureUsage.js - canUseFeature() (mode crédits)

async function canUseFeature(userId, featureName) {
  // Mode crédits uniquement
  const cost = await getCreditCostForFeature(featureName);

  // Feature gratuite ?
  if (cost === 0) {
    return { canUse: true, useCredit: false };
  }

  // Vérifier le solde
  const balance = await getCreditBalance(userId);

  if (balance >= cost) {
    return {
      canUse: true,
      useCredit: true,
      creditCost: cost
    };
  }

  return {
    canUse: false,
    reason: 'insufficient_credits',
    creditCost: cost,
    balance: balance
  };
}

Comparaison des Modèles

Aspect Modèle Abonnement Modèle Crédits

Revenus prévisibles Oui (MRR/ARR) Non
Limites Mensuelles par plan Basées sur solde crédits
Onboarding Plan Free avec limites Crédits de bienvenue
Upgrade path Free → Pro → Premium Achat de plus de crédits
Stripe Products Subscriptions + Prices One-time payments
Webhooks utilisés subscription.*, invoice.* checkout.session.completed
