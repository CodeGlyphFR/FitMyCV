Checkout Flow

L'endpoint POST /api/checkout/subscription gère trois scénarios différents selon l'état de l'abonnement : nouvelle souscription, upgrade avec prorata, ou downgrade programmé.

Scénario 1 : Nouvelle Souscription

Utilisateur sans abonnement Stripe actif (plan Gratuit ou premier achat).

Flux nouvelle souscription
sequenceDiagram participant U as Utilisateur participant App as POST /api/checkout/subscription participant Stripe as Stripe participant WH as Webhooks U->>App: {planId, billingPeriod, locale} App->>App: Vérifier : pas d'abonnement Stripe actif alt Pas de Customer Stripe App->>Stripe: customers.create({email, name, invoice_template}) Stripe-->>App: customer.id App->>App: Mettre à jour User.stripeCustomerId end App->>Stripe: checkout.sessions.create({mode: 'subscription', ...}) Note over Stripe: consent_collection: terms_of_service
allow_promotion_codes: true
billing_address_collection: required Stripe-->>App: session.url App-->>U: Redirection vers Stripe Checkout U->>Stripe: Paiement carte Stripe->>WH: checkout.session.completed WH->>WH: updateCustomerBillingDetailsFromCheckout() Stripe->>WH: customer.subscription.created WH->>App: Créer/MAJ Subscription en DB Stripe->>WH: invoice.paid WH->>App: Reset compteurs features App-->>U: Redirection /account/subscriptions?success=true

Scénario 2 : Upgrade

Utilisateur avec abonnement Stripe actif qui passe à un tier supérieur ou au même tier avec passage mensuel → annuel.

Flux upgrade avec prorata
sequenceDiagram participant U as Utilisateur participant App as POST /api/checkout/subscription participant Stripe as Stripe U->>App: {planId, billingPeriod} App->>Stripe: subscriptions.retrieve(stripeSubscriptionId) App->>App: Déterminer : isUpgrade = true App->>Stripe: subscriptions.update({
items: [{price: newPriceId}],
proration_behavior: 'create_prorations',
billing_cycle_anchor: 'now'
}) Stripe-->>App: updatedSubscription App->>App: changeSubscription(userId, planId, billingPeriod) alt Upgrade Free â†’ Payant App->>App: resetFeatureCounters(userId) end App-->>U: {success: true, upgraded: true}
Prorata automatique

L'upgrade utilise proration_behavior: 'create_prorations' et billing_cycle_anchor: 'now'. Le montant restant de l'ancien plan est crédité automatiquement par Stripe. L'endpoint POST /api/subscription/preview-upgrade permet de calculer le prorata avant d'effectuer l'upgrade.

Scénario 3 : Downgrade

Utilisateur qui passe à un tier inférieur ou au même tier avec passage annuel → mensuel. Le changement est programmé pour la fin de la période en cours.

Flux downgrade programmé
sequenceDiagram participant U as Utilisateur participant App as POST /api/checkout/subscription participant Stripe as Stripe U->>App: {planId, billingPeriod} App->>Stripe: subscriptions.retrieve(stripeSubscriptionId) App->>App: Déterminer : isDowngrade = true alt Schedule existant App->>Stripe: subscriptionSchedules.retrieve(schedule) else Pas de schedule App->>Stripe: subscriptionSchedules.create({from_subscription}) end App->>Stripe: subscriptionSchedules.update({
phases: [
{plan actuel jusqu'à current_period_end},
{nouveau plan après current_period_end}
],
end_behavior: 'release'
}) Stripe-->>App: updatedSchedule App-->>U: {scheduled: true, effectiveDate: "..."} Note over U: L'utilisateur garde le plan actuel
jusqu'à la fin de la période
Annulation de downgrade

POST /api/subscription/cancel-downgrade libère le Subscription Schedule via subscriptionSchedules.release(), annulant le downgrade programmé. Le getSubscriptionSummary() détecte les downgrades programmés en vérifiant si un schedule avec 2 phases est attaché à l'abonnement.

Achat de Packs de Crédits

POST /api/checkout/credits crée une session Stripe Checkout en mode payment (paiement unique).

Flux achat crédits
sequenceDiagram participant U as Utilisateur participant App as POST /api/checkout/credits participant Stripe as Stripe participant WH as Webhooks U->>App: {packId, locale} App->>App: Vérifier pack actif + stripePriceId App->>Stripe: checkout.sessions.create({
mode: 'payment',
metadata: {type: 'credit_purchase', creditAmount}
}) Stripe-->>App: session.url App-->>U: Redirection Stripe U->>Stripe: Paiement Stripe->>WH: checkout.session.completed WH->>App: grantCredits(userId, creditAmount) Stripe->>WH: charge.succeeded WH->>App: Créer facture Stripe WH->>App: Envoyer email confirmation App-->>U: Redirection /account/subscriptions?credits_success=true

Annulation d'Abonnement

POST /api/subscription/cancel supporte deux modes :

Mode Paramètre Comportement
Différé immediate: false Stripe : cancel_at_period_end: true, DB : cancelAtPeriodEnd: true
Immédiat immediate: true Stripe : subscriptions.cancel(), DB : downgrade vers Gratuit
Libération du schedule

Si un Subscription Schedule existe (downgrade programmé), il est libéré automatiquement avant l'annulation pour éviter les conflits.

Preview Prorata

POST /api/subscription/preview-upgrade calcule le montant à payer sans effectuer la modification :

// Utilise stripe.invoices.createPreview()
const upcomingInvoice = await stripe.invoices.createPreview({
  customer: stripeCustomerId,
  subscription: stripeSubscriptionId,
  subscription_details: {
    items: [{ id: itemId, price: newPriceId }],
    proration_behavior: 'create_prorations',
    billing_cycle_anchor: 'now',
  },
});

// Réponse
{
  prorataAmount: 12.50,          // Montant final (après balance)
  prorataAmountBeforeBalance: 15, // Avant déduction solde
  customerBalance: -2.50,        // Crédit client Stripe
  creditAmount: 5.00,            // Crédit de l'ancien plan
  monthsOffered: 0,
  currency: "EUR"
}
