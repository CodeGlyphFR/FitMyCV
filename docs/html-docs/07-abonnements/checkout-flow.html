Checkout Flow

L'endpoint POST /api/checkout/subscription gère trois scénarios différents selon l'état de l'abonnement : nouvelle souscription, upgrade avec prorata, ou downgrade programmé.

Scénario 1 : Nouvelle Souscription

Utilisateur sans abonnement Stripe actif (plan Gratuit ou premier achat).

Codes promo autorisés
Adresse facturation obligatoire Stripe-->>App: URL de paiement App-->>U: Redirection vers Stripe Checkout U->>Stripe: Paiement carte Stripe->>WH: Paiement terminé WH->>WH: Mettre à jour les infos de facturation Stripe->>WH: Abonnement créé WH->>App: Créer/MAJ abonnement en base Stripe->>WH: Facture payée WH->>App: Réinitialiser les compteurs App-->>U: Redirection page abonnements

Scénario 2 : Upgrade

Utilisateur avec abonnement Stripe actif qui passe à un tier supérieur ou au même tier avec passage mensuel → annuel.

Flux upgrade avec prorata
sequenceDiagram participant U as Utilisateur participant App as Serveur participant Stripe as Stripe U->>App: Plan choisi, période App->>Stripe: Récupérer l'abonnement actuel App->>App: Déterminer : upgrade détecté App->>Stripe: Modifier l'abonnement
avec prorata et nouveau prix Stripe-->>App: Abonnement mis à jour App->>App: Mettre à jour l'abonnement en base alt Upgrade Gratuit → Payant App->>App: Réinitialiser les compteurs end App-->>U: Upgrade réussi
Prorata automatique

L'upgrade utilise proration_behavior: 'create_prorations' et billing_cycle_anchor: 'now'. Le montant restant de l'ancien plan est crédité automatiquement par Stripe. L'endpoint POST /api/subscription/preview-upgrade permet de calculer le prorata avant d'effectuer l'upgrade.

Scénario 3 : Downgrade

Utilisateur qui passe à un tier inférieur ou au même tier avec passage annuel → mensuel. Le changement est programmé pour la fin de la période en cours.

Flux downgrade programmé
sequenceDiagram participant U as Utilisateur participant App as Serveur participant Stripe as Stripe U->>App: Plan choisi, période App->>Stripe: Récupérer l'abonnement actuel App->>App: Déterminer : downgrade détecté alt Planning existant App->>Stripe: Récupérer le planning existant else Pas de planning App->>Stripe: Créer un planning depuis l'abonnement end App->>Stripe: Mettre à jour le planning :
plan actuel jusqu'à fin de période,
nouveau plan ensuite Stripe-->>App: Planning mis à jour App-->>U: Downgrade programmé, date effective Note over U: L'utilisateur garde le plan actuel
jusqu'à la fin de la période
Annulation de downgrade

POST /api/subscription/cancel-downgrade libère le Subscription Schedule via subscriptionSchedules.release(), annulant le downgrade programmé. Le getSubscriptionSummary() détecte les downgrades programmés en vérifiant si un schedule avec 2 phases est attaché à l'abonnement.

Achat de Packs de Crédits

POST /api/checkout/credits crée une session Stripe Checkout en mode payment (paiement unique).

Flux achat crédits
sequenceDiagram participant U as Utilisateur participant App as Serveur participant Stripe as Stripe participant WH as Webhooks U->>App: Pack choisi, langue App->>App: Vérifier pack actif et prix Stripe App->>Stripe: Créer une session de paiement unique Stripe-->>App: URL de paiement App-->>U: Redirection Stripe U->>Stripe: Paiement Stripe->>WH: Paiement terminé WH->>App: Attribuer les crédits achetés Stripe->>WH: Charge réussie WH->>App: Créer la facture WH->>App: Envoyer email confirmation App-->>U: Redirection page crédits

Annulation d'Abonnement

POST /api/subscription/cancel supporte deux modes :

Mode Paramètre Comportement
Différé immediate: false Stripe : cancel_at_period_end: true, DB : cancelAtPeriodEnd: true
Immédiat immediate: true Stripe : subscriptions.cancel(), DB : downgrade vers Gratuit
Libération du schedule

Si un Subscription Schedule existe (downgrade programmé), il est libéré automatiquement avant l'annulation pour éviter les conflits.

Localisation Checkout Stripe

Le message d'acceptation des CGV est localisé via lib/stripe/checkoutLocale.js. Les traductions sont extraites des fichiers locales/{lang}/subscription.json :

Preview Prorata

POST /api/subscription/preview-upgrade calcule le montant à payer sans effectuer la modification :

// Utilise stripe.invoices.createPreview()
const upcomingInvoice = await stripe.invoices.createPreview({
  customer: stripeCustomerId,
  subscription: stripeSubscriptionId,
  subscription_details: {
    items: [{ id: itemId, price: newPriceId }],
    proration_behavior: 'create_prorations',
    billing_cycle_anchor: 'now',
  },
});

// Réponse
{
  prorataAmount: 12.50,          // Montant final (après balance)
  prorataAmountBeforeBalance: 15, // Avant déduction solde
  customerBalance: -2.50,        // Crédit client Stripe
  creditAmount: 5.00,            // Crédit de l'ancien plan
  monthsOffered: 0,
  currency: "EUR"
}
