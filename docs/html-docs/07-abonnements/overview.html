Système d'Abonnements

FitMyCV.io utilise Stripe pour gérer les abonnements et paiements. Le système supporte deux modèles économiques configurables : abonnement avec crédits ou crédits uniquement.

Plans d'Abonnement

Les plans sont stockés dans SubscriptionPlan avec un système de tiers (0=Gratuit, 1=Pro, 2=Premium). Les prix et limites sont configurés en base de données et synchronisés avec Stripe.

Plan Tier Prix mensuel Prix annuel Facturation
Gratuit 0 0€ 0€ Pas de Stripe (local uniquement)
Pro 1 Configurable en DB Configurable en DB Stripe Subscription
Premium 2 Configurable en DB Configurable en DB Stripe Subscription
Plans dynamiques

Les prix et caractéristiques des plans ne sont pas hardcodés : ils sont stockés dans la table SubscriptionPlan et synchronisés avec Stripe via lib/subscription/stripeSync.js. Les plans gratuits ne sont jamais sync avec Stripe.

Modèles de Données

SubscriptionPlan

model SubscriptionPlan {
  id                    Int       @id @default(autoincrement())
  name                  String    @unique    // "Gratuit", "Pro", "Premium"
  description           String?
  isFree                Boolean   @default(false)
  tier                  Int       @default(0) // 0=Gratuit, 1=Pro, 2=Premium
  isPopular             Boolean   @default(false)
  priceMonthly          Float     @default(0)
  priceYearly           Float     @default(0)
  yearlyDiscountPercent Float     @default(0)
  priceCurrency         String    @default("EUR")
  stripeProductId       String?   @unique
  stripePriceIdMonthly  String?   @unique
  stripePriceIdYearly   String?   @unique

  subscriptions         Subscription[]
  featureLimits         SubscriptionPlanFeatureLimit[]
}

Subscription

model Subscription {
  id                   String   @id @default(cuid())
  userId               String   @unique
  stripeCustomerId     String   @unique
  stripeSubscriptionId String?  @unique
  stripePriceId        String?
  planId               Int
  status               String    // "active", "canceled", "past_due"
  billingPeriod        String    // "monthly", "yearly"
  currentPeriodStart   DateTime
  currentPeriodEnd     DateTime
  cancelAtPeriodEnd    Boolean  @default(false)
  canceledAt           DateTime?

  plan                 SubscriptionPlan @relation(...)
  user                 User             @relation(...)
}
