Prompts IA

Le pipeline de génération utilise 21 fichiers de prompts organisés par phase. Chaque prompt suit une structure standardisée et respecte la hiérarchie de priorités VÉRITÉ > LANGUE > ATS > STYLE.

Structure Standard d'un Prompt

Chaque prompt suit un template standardisé :

# [Nom de la Mission]

## Contexte
[Description du contexte et de l'objectif]

## Priorités (STRICTES)
1. **VÉRITÉ** (P1) : Jamais d'information inventée
2. **LANGUE** (P2) : Contenu dans la langue cible
3. **ATS** (P3) : Mots-clés exacts de l'offre
4. **STYLE** (P4) : Ton professionnel adapté

## Input
[Description des données en entrée]

## Output
[Description du format de sortie attendu]

## Règles
- [Règle 1]
- [Règle 2]
- ...

## Exemples
### Exemple 1 - [Description]
**Input:**
```json
{ ... }
```
**Output:**
```json
{ ... }
```

## Anti-Patterns
❌ Ne JAMAIS faire : [exemple interdit]
✅ Faire à la place : [exemple correct]

Hiérarchie des Priorités

P1 - VÉRITÉ (Non négociable)

Chaque mot du CV généré DOIT pouvoir être tracé vers le CV source. Aucune information inventée, aucune compétence ajoutée, aucune durée modifiée.

P2 - LANGUE

Tout le contenu doit être dans la langue cible du CV. 4 langues supportées : Français (FR), Anglais (EN), Espagnol (ES), Allemand (DE).

Exceptions (restent inchangés) :

  • Noms propres et entreprises
  • Technologies et frameworks (React, Docker, AWS...)
  • Certifications (AWS Certified, ITIL...)
  • URLs et liens
  • Niveaux CECRL pour les langues (A1-C2)
P3 - ATS (Applicant Tracking System)

Intégrer les mots-clés EXACTS de l'offre d'emploi pour maximiser le score ATS. Utiliser la terminologie de l'offre quand elle correspond à une compétence existante.

P4 - STYLE

Ton professionnel, phrases d'action, format adapté au secteur. Reformulations autorisées tant que le sens est préservé.

Prompts par Phase

Phase Classification

classify-items.md Classification des items
Mission

Analyser chaque expérience et projet du CV source pour décider de son traitement.

Décisions possibles
KEEP Garder et adapter l'item
REMOVE Supprimer (non pertinent)
MOVE_TO_PROJECTS Convertir expérience → projet
Critères de décision
  • Pertinence par rapport au poste cible
  • Ancienneté (> 10 ans = candidat REMOVE)
  • Durée (< 3 mois = candidat MOVE_TO_PROJECTS)
  • Transferabilité des compétences

Phase Batches - Experiences

adapt-experience.md Adaptation d'expérience
Mission

Reformuler une expérience professionnelle pour maximiser sa pertinence vis-à-vis de l'offre cible.

Règles clés
  • Le titre de poste peut être reformulé si une variante existe dans l'offre
  • Les responsabilités sont réordonnées par pertinence
  • Les verbes d'action sont adaptés au secteur
  • Les métriques et chiffres sont CONSERVÉS tels quels
Exemple de reformulation
// AVANT
"Développer des applications web"

// APRÈS (si l'offre mentionne "APIs REST")
"Concevoir et développer des APIs REST et applications web"

// INTERDIT (invention)
"Développer des applications web et mobiles" // mobile non mentionné

Phase Batches - Summary

generate-summary.md Génération du résumé
Mission

Générer un résumé professionnel percutant basé sur les expériences adaptées et l'offre cible.

Structure du résumé
  1. Accroche : Titre + années d'expérience
  2. Expertise : Domaines clés alignés avec l'offre
  3. Réalisations : 1-2 accomplissements majeurs
  4. Valeur : Ce que le candidat apporte
Contraintes
  • 3-5 phrases maximum
  • Intégrer 3-5 mots-clés de l'offre
  • Première personne implicite (pas de "Je")

Génération URLs pour Projets (Templates)

Lors de la génération de CV templates (candidats fictifs), les projets peuvent inclure des URLs cohérentes avec le profil généré.

projects Champs url et url_label
Règles de génération
  • L'URL doit être cohérente avec le header.full_name du candidat fictif
  • Le url_label décrit le type de lien (ex: "GitHub", "Portfolio", "Demo")
  • Les URLs doivent être plausibles mais fictives (domaines existants, usernames générés)
Exemples de sortie
{
  "projects": [
    {
      "name": "E-commerce Platform",
      "url": "https://github.com/marie-dupont/ecommerce-platform",
      "url_label": "GitHub"
    },
    {
      "name": "Portfolio Personnel",
      "url": "https://marie-dupont.dev",
      "url_label": "Portfolio"
    },
    {
      "name": "Application Mobile Fitness",
      "url": "https://play.google.com/store/apps/details?id=com.fitness.app",
      "url_label": "Google Play"
    }
  ]
}
Types de liens générés
GitHub Pour projets open source ou repos privés
Portfolio Site personnel du candidat
Demo Lien vers une démo live
Google Play / App Store Pour applications mobiles
CV Templates uniquement

Cette fonctionnalité s'applique aux CV templates (candidats fictifs générés pour démonstration). Pour les CV réels, les URLs proviennent du CV source et ne sont jamais inventées.

Anti-Hallucination

Règle fondamentale

"Chaque mot doit pointer vers la source"

Type Exemple INTERDIT Raison
Ajout compétence "Python" → "Python/IA" IA non mentionné dans la source
Modification durée "2 ans" → "3 ans" Falsification
Ajout métrique "Amélioration" → "Amélioration de 30%" Chiffre inventé
Invention responsabilité Ajouter "Management d'équipe" Non présent dans la source

Traçabilité des Modifications

Chaque prompt demande de documenter les modifications :

{
  "modifications": [
    {
      "field": "title",
      "action": "reformulated",
      "before": "Développeur Web",
      "after": "Développeur Full Stack",
      "reason": "Alignement avec titre de l'offre"
    },
    {
      "field": "responsibilities[0]",
      "action": "reworded",
      "before": "Création d'applications",
      "after": "Développement d'APIs REST",
      "reason": "Mise en avant skill requis"
    }
  ]
}

Chargement des Prompts

// lib/features/cv-adaptation/utils/promptLoader.js
import fs from 'fs/promises';
import path from 'path';

const PROMPTS_DIR = path.join(process.cwd(), 'lib/features/cv-adaptation/prompts');

export async function loadPrompt(promptPath) {
  const fullPath = path.join(PROMPTS_DIR, `${promptPath}.md`);
  const content = await fs.readFile(fullPath, 'utf-8');
  return content;
}

// Utilisation
const prompt = await loadPrompt('experiences/adapt-experience');
const messages = [
  { role: 'system', content: prompt },
  { role: 'user', content: JSON.stringify(inputData) }
];
