Pipeline de Génération CV

Le pipeline de génération CV (cv-adaptation) transforme automatiquement un CV source pour l'adapter à une offre d'emploi spécifique. Il utilise une architecture par phases avec parallélisation et cache intelligent.

Coût

Le nombre de crédits par génération est configurable dans Administration → Settings. Le crédit est débité au lancement et remboursé en cas d'échec.

Étape 0 : Extraction

Fichier : lib/features/cv-adaptation/phases/extract.js

EXTRACTION - Récupération de l'offre d'emploi
Données d'entrée

Sources possibles :

  • URL : Page web d'annonce d'emploi
  • PDF : Document uploadé par l'utilisateur
  • Cache : JobOffer déjà extraite (réutilisation)
Résumé du prompt
"Expert en extraction d'offres d'emploi. Analyser le contenu de la page/document et extraire les informations structurées : titre, entreprise, compétences requises et souhaitées, responsabilités, avantages."
Format de sortie (JobOffer)
{
  "title": "Senior Full-Stack Developer",
  "company": "TechCorp",
  "location": "Paris, France",
  "contract_type": "CDI",
  "skills": {
    "required": ["React", "Node.js", "TypeScript", "PostgreSQL"],
    "nice_to_have": ["GraphQL", "Docker", "AWS"]
  },
  "soft_skills": ["Leadership", "Communication", "Problem-solving"],
  "methodologies": ["Agile", "Scrum", "CI/CD"],
  "responsibilities": [
    "Développer des features full-stack",
    "Mentorer les développeurs juniors",
    "Participer aux code reviews"
  ],
  "benefits": ["Remote friendly", "RTT", "Mutuelle"]
}

Étape 0.5 : Classification

Fichier : lib/features/cv-adaptation/phases/classify.js

CLASSIFICATION - Tri des expériences et projets
Données d'entrée

Provenance : CvFile.content + JobOffer

{
  "cvExperiences": [
    { "title": "Lead Developer", "company": "StartupX", ... },
    { "title": "Serveur Restaurant", "company": "ChezPaul", ... }
  ],
  "cvProjects": [
    { "name": "E-commerce Platform", ... }
  ],
  "jobOffer": { "title": "Senior Full-Stack Developer", ... }
}
Résumé du prompt
"Expert en recrutement. Classifier chaque expérience et projet du CV selon leur pertinence pour l'offre d'emploi. Décisions possibles : KEEP (pertinent), REMOVE (non pertinent), MOVE_TO_PROJECTS (expérience à convertir en projet)."

Critères de décision :

  • KEEP : Expérience/projet aligné avec le poste
  • REMOVE : Sans rapport (ex: job étudiant non tech pour poste dev)
  • MOVE_TO_PROJECTS : Mission courte ou freelance pertinente
Format de sortie
{
  "experiences": [
    {
      "index": 0,
      "title": "Lead Developer @ StartupX",
      "decision": "KEEP",
      "reason": "Expérience full-stack pertinente avec leadership"
    },
    {
      "index": 1,
      "title": "Serveur @ ChezPaul",
      "decision": "REMOVE",
      "reason": "Non pertinent pour poste technique"
    }
  ],
  "projects": [
    {
      "index": 0,
      "name": "E-commerce Platform",
      "decision": "KEEP",
      "reason": "Stack technique aligné"
    }
  ]
}

Étape 1 : Batches (Parallèle)

Tous les batches s'exécutent en parallèle grâce au système de cache CAG.

1a: Batch Experiences

Adapter chaque expérience classée KEEP :

  • Reformuler responsibilities
  • Mettre en avant les deliverables
  • Ajouter mots-clés de l'offre
  • Respecter la langue cible

Modèle : gpt-4o

1b: Batch Projects

Adapter projets existants + convertir MOVE_TO_PROJECTS :

  • Convertir expériences en projets
  • Mettre en avant les technologies
  • Ajouter résultats quantifiés

Modèle : gpt-4o

1c: Batch Extras

Adapter les sections complémentaires :

  • Certifications
  • Bénévolat
  • Publications
  • Centres d'intérêt pertinents

Modèle : gpt-4o-mini

1d: Batch Skills

7 missions d'adaptation des compétences :

  • hard_skills, soft_skills
  • tools, methodologies
  • certifications, interests
  • languages

Modèle : gpt-4o-mini

1e: Batch Summary

Générer ou adapter le résumé professionnel :

  • Synthétiser le parcours
  • Mettre en avant l'adéquation avec le poste
  • Intégrer les mots-clés stratégiques
  • 3-4 phrases percutantes

Modèle : gpt-4o

BATCH EXPERIENCES - Exemple d'adaptation
Données d'entrée
{
  "experience": {
    "title": "Lead Developer",
    "company": "StartupX",
    "responsibilities": ["Développer des applications"],
    "deliverables": [],
    "skills_used": ["JavaScript"]
  },
  "jobOffer": {
    "skills": { "required": ["React", "Node.js", "TypeScript"] }
  },
  "cacheA": { /* contexte offre */ }
}
Format de sortie
{
  "adapted_experience": {
    "title": "Lead Developer",
    "company": "StartupX",
    "responsibilities": [
      "Concevoir et développer des applications web React/Node.js",
      "Définir l'architecture technique et les bonnes pratiques"
    ],
    "deliverables": [
      "Application SaaS servant 10K+ utilisateurs",
      "Réduction de 40% du temps de chargement"
    ],
    "skills_used": ["React", "Node.js", "TypeScript", "PostgreSQL"]
  },
  "modifications": [
    {
      "field": "responsibilities",
      "action": "modified",
      "before": "Développer des applications",
      "after": "Concevoir et développer des applications web React/Node.js",
      "reason": "Alignement avec stack technique de l'offre"
    }
  ]
}

Étape 2 : Recomposition

Fichier : lib/features/cv-adaptation/phases/recompose.js

RECOMPOSITION - Assemblage du CV final
Données d'entrée

Provenance : Résultats de tous les batches

{
  "classification": { /* décisions KEEP/REMOVE/MOVE */ },
  "adaptedExperiences": [ /* expériences adaptées */ ],
  "adaptedProjects": [ /* projets adaptés */ ],
  "adaptedExtras": { /* sections extras */ },
  "adaptedSkills": { /* toutes les compétences */ },
  "adaptedSummary": "Résumé professionnel...",
  "originalCv": { /* CV source pour header, education, languages */ }
}
Processus
Cette phase est purement algorithmique (pas d'appel IA). Elle assemble les résultats des batches en respectant l'ordre et la structure du CV.

Étapes d'assemblage :

  1. Copier header et education du CV source
  2. Insérer le summary adapté
  3. Filtrer experiences (exclure REMOVE, réordonner)
  4. Fusionner projects (existants + convertis)
  5. Assembler skills par catégorie
  6. Copier languages du CV source
  7. Intégrer extras adaptés
CV Final
{
  "header": { /* inchangé */ },
  "summary": "Développeur Full-Stack avec 8 ans d'expérience...",
  "experience": [ /* expériences adaptées, ordonnées */ ],
  "projects": [ /* projets fusionnés */ ],
  "skills": {
    "hard_skills": [...],
    "soft_skills": [...],
    "tools": [...],
    "methodologies": [...]
  },
  "education": { /* inchangé */ },
  "languages": { /* inchangé */ },
  "extras": { /* adaptés */ }
}

Système de Cache (CAG)

Le pipeline utilise un système de cache à deux niveaux pour optimiser les coûts OpenAI et garantir la cohérence :

Cache A

Utilisé par : Experiences, Projects, Extras

  • Job offer title
  • Required skills
  • Nice-to-have skills
  • Soft skills
  • Methodologies
  • Responsibilities (top 5)

Cache B

Utilisé par : Skills, Summary

  • Tout le Cache A
  • + Expériences adaptées (résumé)
  • + Projets adaptés (résumé)
Optimisation des coûts

Les tokens du cache (cache_read_input_tokens) coûtent 90% moins cher que les tokens standards. Le cache est maintenu pendant 5 minutes (dev) ou 24h (GPT-5).

Hiérarchie des Priorités

Tous les prompts respectent cette hiérarchie stricte :

Priorité Nom Description
P1 VÉRITÉ Jamais d'information inventée - traçabilité obligatoire
P2 LANGUE Contenu dans la langue cible du CV
P3 ATS Mots-clés exacts de l'offre d'emploi
P4 STYLE Ton professionnel et format adapté
Anti-Hallucination

Règle fondamentale : "Chaque mot doit pointer vers la source". Les modifications sont des reformulations, jamais des inventions. Exemple interdit : "Développeur Python" → "Développeur Python/IA" (IA n'existe pas dans la source).

Traçabilité et Modifications

Chaque batch retourne un tableau modifications documentant chaque changement :

{
  "modifications": [
    {
      "field": "responsibilities",
      "action": "modified",
      "before": "Développer des applications",
      "after": "Concevoir et développer des APIs REST",
      "reason": "Alignement avec requirement 'API REST' de l'offre"
    }
  ]
}

Différences avec Pipeline Optimisation

Aspect Pipeline Génération Pipeline Optimisation
Objectif Transformation complète Amélioration ciblée
Offre d'emploi Obligatoire Optionnelle
Classification IA classe automatiquement Utilisateur choisit
Review Optionnel (post-génération) Intégré au workflow
Coût 1 crédit (configurable) 2 crédits (+ 1 scoring)
Cache Cache A/B structuré Pas de cache spécifique
