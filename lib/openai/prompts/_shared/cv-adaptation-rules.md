# REGLES D'ADAPTATION DE CV

## 1. COMPETENCES

### CATEGORIES (MUTUELLEMENT EXCLUSIVES)

Chaque competence doit appartenir a UNE SEULE categorie. Ne jamais dupliquer.

| Categorie | Definition | Exemples | CE QUE CE N'EST PAS |
|-----------|------------|----------|---------------------|
| **hard_skills** | Savoir-faire technique, domaine d'expertise | JavaScript, Machine Learning, Data Analysis, SEO, Comptabilite, UX Design | Pas un logiciel, pas un outil |
| **soft_skills** | Qualites comportementales, savoir-etre | Leadership, Communication, Travail en equipe, Adaptabilite, Resolution de problemes | Pas une competence technique |
| **tools** | Logiciels, applications, plateformes | VS Code, Figma, Excel, Docker, AWS, Jira, Slack, Photoshop | Pas un domaine d'expertise |
| **methodologies** | Methodes de travail, frameworks organisationnels | Agile, Scrum, Kanban, Lean, DevOps, CI/CD, TDD | Pas un outil, pas une competence technique |

### REGLE ANTI-DUPLICATION

AVANT d'ajouter une competence, verifier :
1. Est-ce un LOGICIEL/APPLICATION qu'on installe ? → **tools**
2. Est-ce une METHODE DE TRAVAIL/ORGANISATION ? → **methodologies**
3. Est-ce une QUALITE PERSONNELLE/COMPORTEMENT ? → **soft_skills**
4. Est-ce un DOMAINE D'EXPERTISE TECHNIQUE ? → **hard_skills**

**Exemples de classification correcte :**
- Docker → **tools** (c'est un logiciel)
- Conteneurisation → **hard_skills** (c'est un domaine)
- Git → **tools** (c'est un logiciel)
- Gestion de version → NE PAS AJOUTER (redondant avec Git)
- Excel → **tools**
- Analyse de donnees → **hard_skills** (si justifie par le CV)
- Scrum → **methodologies**
- Gestion de projet → **hard_skills** (domaine d'expertise)

### A FAIRE
- SUPPRIMER les competences hors sujet par rapport a l'offre d'emploi
- AJOUTER des competences UNIQUEMENT SI :
  1. Demandees dans l'offre d'emploi ET
  2. Justifiables par les experiences, formations ou projets du CV
  3. PAS deja presentes dans une autre categorie
- EVALUER le niveau selon les preuves du CV :
  - `Awareness` : exposition courte, projets academiques
  - `Beginner` : ~1 an d'experience
  - `Intermediate` : 2-3 ans
  - `Proficient` : applique en projet reel
  - `Advanced` : 3-7 ans
  - `Expert` : 7+ ans
- REORGANISER par pertinence (plus pertinent en premier)

### A NE PAS FAIRE
- Mettre la meme competence dans plusieurs categories
- Confondre un outil (Docker) avec son domaine (Conteneurisation)
- Inventer des competences non supportees par le CV
- Ajouter des certifications non obtenues

## 2. EXPERIENCES

### REGLE DE CONDENSATION

**MAXIMUM 5 BULLETS** par experience (responsibilities + deliverables combines).

Pour condenser :
1. IDENTIFIER les responsabilites pertinentes pour l'offre d'emploi
2. FUSIONNER les bullets similaires ou redondantes
3. SUPPRIMER les details non pertinents pour le poste vise
4. GARDER uniquement l'information UTILE pour repondre a l'offre

**Exemple de condensation :**
```
AVANT (8 bullets) :
- Developper des fonctionnalites frontend
- Creer des composants React
- Ecrire des tests unitaires
- Participer aux code reviews
- Corriger des bugs
- Documenter le code
- Assister aux daily meetings
- Former les nouveaux developpeurs

APRES (4 bullets - pour une offre React Senior) :
- Developper des composants React reutilisables avec tests unitaires
- Effectuer des code reviews et former les nouveaux developpeurs
- Corriger les bugs et optimiser les performances frontend
- Documenter l'architecture et les patterns utilises
```

### A FAIRE
- REFORMULER chaque bullet avec le format :
  `[Verbe d'action] + [Tache precise] + [Resultat chiffre si disponible]`
- Verbes recommandes : Piloter, Concevoir, Developper, Optimiser, Deployer, Coordonner, Analyser, Reduire, Augmenter, Creer, Implementer
- Max 10-15 mots par bullet
- Utiliser les termes exacts de l'offre (optimisation ATS)
- CONSERVER toutes les experiences (ne pas en supprimer)
- CONDENSER les bullets a 5 max par experience
- INCLURE les resultats quantifies SEULEMENT s'ils sont presents ou deductibles du CV source

### A NE PAS FAIRE
- Depasser 5 bullets par experience
- Inventer des experiences ou responsabilites
- Ajouter des metriques fictives
- Utiliser des constructions passives ("Responsable de" -> "Piloter")
- Reorganiser les experiences (ordre chronologique impose)
- Supprimer des informations importantes pour l'offre

## 3. RESUME

### A FAIRE
- Ecrire 1-2 phrases presentant le candidat
- Personnaliser selon l'offre tout en restant coherent avec le CV
- Utiliser le vocabulaire de l'offre

### A NE PAS FAIRE
- Inventer des expertises non supportees par le CV

---

## 4. HEADER

### A FAIRE
- Adapter `current_title` au titre du poste vise
- Garder coherent avec le niveau d'experience reel

### A NE PAS FAIRE
- Modifier les coordonnees, nom ou liens

---

## 5. NE PAS TOUCHER

Les sections suivantes doivent rester inchangees :
- `education` (diplomes, ecoles, dates)
- `languages` (sauf reordonnancement si une langue est prioritaire)
- `projects`
- `extras` (certifications, hobbies)
- Informations de contact

---

## 6. STYLE D'ECRITURE

### UTILISER
- Ton direct et professionnel
- Affirmations factuelles plutot qu'adjectifs
- Voix active
- Phrases courtes (5-15 mots)
- Chiffres specifiques quand disponibles

### EVITER
- Buzzwords : synergie, disruptif, proactif, impactant, holistique, robuste, levier, ecosysteme, paradigme
- Auto-proclamations : expert en, specialiste de, passionne de
- Superlatifs : irreprochable, exceptionnel, hors pair
- Cliches : force de proposition, valeur ajoutee, sortir des sentiers battus
- Ton trop enthousiaste ou commercial
- Metaphores sportives ou guerrieres

**Regle d'or : Ecrire comme un humain - naturel, factuel, sans exageration.**

---

## 7. VALIDATION

Avant de retourner le resultat, verifier :
- Chaque modification est defendable en entretien
- Aucun contenu invente
- Ton professionnel mais humain
- Mots-cles ATS de l'offre integres naturellement
- Niveaux de competences coherents avec les preuves du CV
