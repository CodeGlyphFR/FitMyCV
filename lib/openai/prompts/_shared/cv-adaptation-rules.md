# REGLES D'ADAPTATION DE CV

---

## 1. REGLES PAR PROFIL

**Detection automatique :**
- Junior : < 3 ans d'experience ou jeune diplome
- Confirme : 3-10 ans avec postes a responsabilite
- Senior : > 10 ans ou postes de direction

### JUNIOR

**Priorites :** Formation detaillee, stages/alternances, projets academiques

**Template accroche :**
> "[Diplome] avec [X] experience(s) en [domaine]. Competences en [skill 1, 2, 3]. Recherche [objectif]."

### CONFIRME

**Priorites :** Experiences avec resultats chiffres, evolution de carriere

**Template accroche :**
> "[X] ans d'experience en [domaine], specialise en [expertise]. [Realisation majeure]. Objectif : [ambition]."

### SENIOR

**Priorites :** Synthese experiences anciennes, focus 10-15 dernieres annees, impact strategique

**Template accroche :**
> "[Titre] avec [X]+ ans d'experience. Expert en [domaines]. [Impact strategique majeur]."

## 2. COMPETENCES

### REGLE CRITIQUE - PERTINENCE SEMANTIQUE

Avant de supprimer une competence, verifier si elle appartient au **MEME DOMAINE** que l'offre.

| Domaine de l'offre | Competences a TOUJOURS GARDER |
|--------------------|-------------------------------|
| IA / GenAI / ML | OpenAI API, Claude API, LLM, Agents IA, Python, Prompt Engineering |
| DevOps / Cloud | Docker, Kubernetes, Linux, CI/CD, administration systeme |
| Developpement | Git, langages de programmation, frameworks, Linux |

**NE JAMAIS supprimer une competence du meme domaine que l'offre, meme si elle n'est pas mentionnee textuellement.**

### Classification stricte (mutuellement exclusive)

| Categorie | Definition | Exemples |
|-----------|------------|----------|
| hard_skills | Domaine d'expertise | JavaScript, Data Analysis, SEO, Comptabilite, Gestion de projet |
| soft_skills | Qualite comportementale | Leadership, Communication, Autonomie, Adaptabilite |
| tools | Logiciel qu'on installe | VS Code, Docker, Excel, Figma, Jira, AWS |
| methodologies | Framework/methode formelle | Agile, Scrum, CI/CD, Lean, ISO 9001 |

### Erreurs frequentes

| ERREUR | CORRECTION |
|--------|------------|
| Autonomie → methodologies | → soft_skills |
| Communication → methodologies | → soft_skills |
| Gestion de projet → methodologies | → hard_skills (domaine) |
| Docker → hard_skills | → tools (logiciel) |
| **Supprimer OpenAI API pour poste GenAI** | → **NE JAMAIS FAIRE** |
| **Supprimer Git pour poste dev** | → **NE JAMAIS FAIRE** |

### Logiciels par categorie (tous → tools)

- **Bureautique :** Excel, Word, PowerPoint, Google Sheets
- **CAO :** CATIA, SolidWorks, AutoCAD
- **Design :** Photoshop, Figma, Sketch
- **Dev :** VS Code, Git, Docker, Jenkins
- **Cloud :** AWS, Azure, GCP
- **IA/LLM :** OpenAI API, Claude API, Cursor, Copilot

**Regle simple :** Si c'est un logiciel qu'on installe → tools

### Niveaux

| Niveau | Critere |
|--------|---------|
| Beginner | ~1 an ou projets academiques |
| Intermediate | 2-3 ans, role secondaire |
| Proficient | Contexte professionnel regulier |
| Advanced | 3-7 ans, resultats demontres |
| Expert | 7+ ans, reference dans le domaine |

## 3. EXPERIENCES

### Format des bullet points

```
[Verbe infinitif] + [Tache precise] + [Resultat chiffre si disponible]
```

**Maximum 5 bullets** par experience.

### Verbes d'action

| Domaine | Verbes |
|---------|--------|
| Tech | Developper, Concevoir, Deployer, Optimiser |
| Management | Piloter, Coordonner, Encadrer, Former |
| Business | Negocier, Generer, Augmenter, Reduire |
| Analyse | Analyser, Auditer, Evaluer, Mesurer |

### Transformation

| AVANT | APRES |
|-------|-------|
| Responsable du developpement | Developper l'interface React (15 ecrans) |
| J'ai gere une equipe | Encadrer 5 developpeurs, +30% productivite |
| Travail sur les performances | Optimiser les requetes SQL, temps /3 |

### Regles

**A FAIRE :**
- Reformuler avec vocabulaire de l'offre
- Condenser a 4-5 bullets pertinents
- Inclure resultats chiffres SI presents dans le CV source

**A NE PAS FAIRE :**
- Inventer des responsabilites
- Ajouter des metriques fictives
- Garder des bullets hors-sujet

## 4. ANTI-HALLUCINATION

### Regle critique

Toute information DOIT etre :
1. Presente dans le CV source, OU
2. Directement deductible des experiences

### Deductions valides

| Si le CV mentionne... | Tu peux deduire... |
|-----------------------|-------------------|
| Management de 5 personnes | Leadership |
| Entreprise internationale | Anglais professionnel |
| Developpement d'API REST | API Design |

### Deductions invalides

- Ajouter "Python" car le candidat est developpeur → NON
- Inventer un pourcentage d'amelioration → NON
- Attribuer "Expert" sans preuve → NON

**Regle d'or :** Chaque ligne doit etre defendable en entretien.

---

## 5. STYLE D'ECRITURE

### Utiliser

- Ton direct et professionnel
- Voix active, phrases courtes (5-15 mots)
- Chiffres specifiques quand disponibles

### Eviter

| Categorie | Exemples a bannir |
|-----------|-------------------|
| Buzzwords | synergie, disruptif, proactif |
| Auto-proclamations | expert en, passionne de |
| Superlatifs | exceptionnel, irreprochable |
| Cliches | force de proposition, valeur ajoutee |

---

## 6. SECTIONS A NE PAS TOUCHER

- education (sauf reordonnancement)
- languages (sauf reordonnancement)
- projects
- extras
- Informations de contact
